## 1. Create the workflow file

Create a file called `hello.duck.yaml` with the following content:

```yaml
flow:
  - type: exec
    run: echo "Hello, duckflux!"
```

That's it — one step, one command. A workflow is just a YAML file with a `flow` field.

---

## 2. Install the runtime

Duckflux workflows are plain YAML files — they need a **runtime** to execute. Install it globally:

```bash
npm install -g duckflux
```

<Aside>
  For more installation options (Bun, Homebrew, standalone binaries, etc.), see the full [CLI installation guide](/runtime/cli/#installation).
</Aside>

---

## 3. Run the workflow

Once the runtime is installed, run your workflow:

```sh
quack run hello.duck.yaml
```

The runtime reads the file, executes the flow, and prints the output:

```
Hello, duckflux!
```

---

<Aside type="tip">
  Workflows support much more — parallel execution, loops, conditionals, error handling, retry, timeouts, and nested sub-workflows. See [Workflow files](/workflows/) for the full reference.
</Aside>

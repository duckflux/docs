Duckflux workflows are plain YAML files — they don't run on their own. You need a **runtime** to execute them.

The official runtime is written in TypeScript and targets Node.js and Bun.

<LinkCard title="Runtime" href="/runtime/cli/" description="Install and use the duckflux CLI." />

---

## Install the runtime

See the [CLI installation guide](/runtime/cli/#installation) for setup instructions.

---

## Run a workflow

Once the runtime is installed, use `quack run` to run any `.duck.yaml` file:

```sh
quack run hello.duck.yaml
```

That's the full command. The runtime reads the file, executes the flow, and prints the output.

### Specifying a working directory

Use `--cwd` to set the base working directory for `exec` participants:

```sh
quack run deploy.duck.yaml --cwd ./repo
```

This is equivalent to setting `defaults.cwd` in the workflow file itself, but applied from the command line. Useful when the workflow is shared but the working directory varies per environment.

<Aside type="tip">
  The `--cwd` flag sets the base directory. Individual participants can still override it with their own `cwd` field. See [Working directory](/runtime/working-directory/) for the full precedence rules.
</Aside>
